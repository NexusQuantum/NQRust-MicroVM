## What are Function Logs?

Function logs capture all output from your function executions, including:

- ‚úÖ **Console output** - `console.log()`, `print()` statements
- ‚úÖ **Error messages** - Stack traces and exceptions
- ‚úÖ **Execution metadata** - Request ID, duration, status code
- ‚úÖ **System messages** - Cold starts, timeouts, crashes

![Image: Logs page overview](/images/functions/logs-overview.png)

**Use logs to**:
- Debug function errors
- Monitor function behavior
- Trace execution flow
- Analyze performance
- Audit invocations

## Log Entry Format

Each log entry shows:

![Image: Single log entry breakdown](/images/functions/log-entry-format.png)

**Components**:

1. **Timestamp** - When the log was created
   - Format: `2025-12-18 14:32:05.123`
   - Timezone: Your local time

2. **Invocation ID** - Unique identifier for this execution
   - Example: `inv-abc123def456`
   - Click to filter logs for this invocation only

3. **Log Level** - Severity indicator
   - INFO (blue) - Normal operation
   - WARN (yellow) - Warnings
   - ERROR (red) - Errors
   - DEBUG (gray) - Debug messages

4. **Message** - The actual log content
   - Console output
   - Error messages
   - System messages

5. **Metadata** - Additional context
   - Duration (e.g., "245ms")
   - Status code (e.g., "200")
   - Memory usage

## Log Types

### Console Output Logs

Logs from your code's console/print statements:

![Image: Console output logs](/images/functions/logs-console-output.png)

**Python**:
```python
def handler(event):
    print("Processing event:", event)  # Shows in logs
    print(f"User ID: {event.get('userId')}")
    return {"statusCode": 200, "body": "OK"}
```

**JavaScript/TypeScript**:
```typescript
export async function handler(event) {
  console.log("Processing event:", event);  // Shows in logs
  console.log(`User ID: ${event?.userId}`);
  return { statusCode: 200, body: "OK" };
}
```

**Example log output**:
```
2025-12-18 14:32:05.123  INFO  [inv-abc123]  Processing event: { userId: 123 }
2025-12-18 14:32:05.125  INFO  [inv-abc123]  User ID: 123
```

Logs when your function throws an error:

### Search Logs

Search for specific text in log messages:

![Image: Search box in logs page](/images/functions/logs-search-box.png)

**How to use**:
1. Enter search term in the search box
2. Logs are filtered in real-time

**Search examples**:

- `inv-abc123` - Find specific invocation

**Case-insensitive**: Search works regardless of capitalization.

### Tracing Execution Flow

Add logging at key points in your code:

**Python**:
```python
def handler(event):
    print("[START] Processing event")

    user_id = event.get("userId")
    print(f"[STEP 1] Extracted userId: {user_id}")

    if not user_id:
        print("[ERROR] Missing userId")
        return {"statusCode": 400, "body": "userId required"}

    result = process_user(user_id)
    print(f"[STEP 2] Processed user, result: {result}")

    print("[SUCCESS] Returning response")
    return {"statusCode": 200, "body": str(result)}
```

**Log output**:
```
2025-12-18 15:00:00.123  INFO  [inv-xyz789]  [START] Processing event
2025-12-18 15:00:00.125  INFO  [inv-xyz789]  [STEP 1] Extracted userId: 123
2025-12-18 15:00:00.230  INFO  [inv-xyz789]  [STEP 2] Processed user, result: {...}
2025-12-18 15:00:00.232  INFO  [inv-xyz789]  [SUCCESS] Returning response
```

**Benefits**:
- See execution order
- Identify where function stops
- Track variable values
- Measure step duration

### Performance Analysis

Use logs to identify performance bottlenecks:

**Add timing logs**:
```typescript
export async function handler(event) {
  const startTime = Date.now();
  console.log("[PERF] Function started");

  const data = await fetchData(event.url);
  console.log(`[PERF] Data fetched in ${Date.now() - startTime}ms`);

  const processed = processData(data);
  console.log(`[PERF] Data processed in ${Date.now() - startTime}ms`);

  console.log(`[PERF] Total execution: ${Date.now() - startTime}ms`);
  return { statusCode: 200, body: JSON.stringify(processed) };
}
```

**Log output**:
```
2025-12-18 15:10:00.100  INFO  [inv-abc123]  [PERF] Function started
2025-12-18 15:10:01.450  INFO  [inv-abc123]  [PERF] Data fetched in 1350ms
2025-12-18 15:10:01.500  INFO  [inv-abc123]  [PERF] Data processed in 1400ms
2025-12-18 15:10:01.505  INFO  [inv-abc123]  [PERF] Total execution: 1405ms
```

**Analysis**: Data fetching takes 96% of execution time ‚Üí Optimize API call or add caching.

## Troubleshooting

### Issue: No Logs Appearing

**Symptoms**:
- Logs page is empty
- No logs after invocation

![Image: Empty logs page](/images/functions/troubleshoot-no-logs.png)

**Solutions**:
1. **Check time range** - Expand to "Last 24 hours"
2. **Check filters** - Clear all filters (click "Clear Filters")
3. **Verify invocation** - Ensure function was actually invoked
4. **Refresh page** - Click refresh button or press F5
5. **Wait a moment** - Logs may take 1-2 seconds to appear

---

### Issue: Logs Cut Off

**Symptoms**:
- Stack trace incomplete
- Long messages truncated

**Solutions**:
1. **Click "Show More"** on the log entry to expand
2. **Export logs** - Full logs are included in exports
3. **Reduce log verbosity** - Split large messages into smaller logs

---

### Issue: Can't Find Specific Invocation

**Symptoms**:
- Looking for logs from specific test
- Too many invocations to browse

**Solutions**:
1. **Use invocation ID** from invoke dialog:
   - Invoke response shows invocation ID
   - Search for the ID in logs
2. **Use time range filter**:
   - Note the time you invoked
   - Filter to that specific time range
3. **Use search**:
   - Search for unique text from your payload
   - Example: If you sent `{"orderId": "ABC123"}`, search "ABC123"

---

### Issue: Logs Loading Slowly

**Symptoms**:
- Logs page takes long to load
- Browser freezes

**Solutions**:
1. **Reduce time range** - Use "Last 5 minutes" instead of "Last 7 days"
2. **Apply filters** - Filter by log level or search term
3. **Clear browser cache** - Old cached data may cause issues
4. **Check network** - Slow connection affects load time

## Best Practices

### Logging Strategy

‚úÖ **Log important events**:
```typescript
console.log("User authenticated:", userId);
console.log("Database query completed");
console.log("Email sent successfully");
```

‚úÖ **Log errors with context**:
```typescript
try {
  await processPayment(data);
} catch (error) {
  console.error("Payment processing failed:", error);
  console.error("Payment data:", data);
  throw error;
}
```

‚úÖ **Use log levels appropriately**:
```python
print("INFO: Normal operation")  # Regular flow
print("WARN: Retrying failed request")  # Warning
print("ERROR: Database connection failed")  # Error
```

‚ùå **Avoid excessive logging**:
```typescript
// Too much - logs on every loop iteration
for (let i = 0; i < 1000; i++) {
  console.log(`Processing item ${i}`);  // 1000 log entries!
}

// Better - log summary
console.log(`Processing ${items.length} items`);
console.log(`Completed processing`);
```

‚ùå **Don't log sensitive data**:
```python
# Bad - logs password
print(f"User login: {username}, password: {password}")

# Good - log without sensitive data
print(f"User login attempt: {username}")
```

### Debugging Workflow

**Step 1: Reproduce the issue**
- Invoke function with same payload
- Note the invocation ID

**Step 2: View logs**
- Filter to that invocation
- Identify error message

**Step 3: Add debug logs**
- Update function code with additional logging
- Deploy updated function

**Step 4: Test again**
- Invoke with same payload
- View new logs with debug info

**Step 5: Fix and verify**
- Fix the issue
- Test with multiple payloads
- Verify logs show success

### Performance Monitoring

Track execution time:

```python
import time

def handler(event):
    start = time.time()

    result = expensive_operation()

    duration = (time.time() - start) * 1000
    print(f"[PERF] Operation completed in {duration:.2f}ms")

    return {"statusCode": 200, "body": str(result)}
```

Set performance budgets:

```typescript
const PERF_BUDGET_MS = 500;

export async function handler(event) {
  const start = Date.now();

  const result = await process(event);

  const duration = Date.now() - start;
  if (duration > PERF_BUDGET_MS) {
    console.warn(`[PERF] Slow execution: ${duration}ms (budget: ${PERF_BUDGET_MS}ms)`);
  }

  return { statusCode: 200, body: JSON.stringify(result) };
}
```
